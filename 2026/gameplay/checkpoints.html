Checkpoints

IMPORTANT! This module requires the Trigger Setup and Hazard Setup modules to be completed first!

Checkpoints are points in a level where the player can effectively save their progress, allowing them to return to the checkpoint if they fail instead of restarting the whole level! Adding checkpoints can allow you to make longer and more challenging levels while also lessening how "unfair" the game feels. On this page we will create a checkpoint that activates when the player touches it, allowing them to respawn and playing a short animation.

Creating the Checkpoint Scene

First things first, we need to create a new scene. First find your Trigger scene in your file system, right click it and select New Inherited Scene.

Next, go ahead and rename the root node of the new scene to Checkpoint. The scene tree should now look like this:

Creating the Checkpoint Sprite Frames and Collision Shape

Next select the AnimatedSprite2D node. In the inspector create a new sprite frames. Import the Checkpoint sprite from the Checkpoints folder in your VGDC Assets folder. Choose the color of checkpoint you want and import it as a sprite sheet.

We can keep the name default for the animation. Additionally, lets disable autoplay and looping, and set the animation speed to 12.

Next select the CollisionShape2D node in the scene tree. In the inspector create a new rectangle collision shape. We want the new shape to be slightly larger than the checkpoint itself (you can change the animation frame to the last one in the inspector if you want to see the full size of the flower to set the collision shape, just make sure to set it back to 0 after!)

Creating the Checkpoint Script

Of course we still don't have any functionality yet. Let's add a new script to the Checkpoint node. First, disconnect the Trigger script and create a new one inheriting our Trigger class.

In the checkpoint script itself we need to first add a new variable: active. This variable will store whether or not the checkpoint has been activated, in order to prevent it from playing it's animation each time the player touches it. Let's start it off as false.

var active = false

Finally, let's override the on_enter function from the Trigger class, having it set the player's respawn_point, play its animation (if not already active) and the inherited on_enter function, then finally setting its active variable to true:

func on_enter(player):
	player.respawn_point = global_position
	if not active:
		$AnimatedSprite2D.play()
		super.on_enter(player)
	active = true

And you are done! Go ahead and add some checkpoints into your world and try them out, then move on to another module!
