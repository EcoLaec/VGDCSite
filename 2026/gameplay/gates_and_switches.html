Gates and Switches

Locked doors, levers, keycards, buttons, sealed gates. Games love their closed doors and convoluted ways to open them. In this module we will create a gate and levers to give you an extra tool in designing your level and encourage exploration.

IMPORTANT! This module requires the Trigger Setup module to be completed first!

Creating the Lever Scene

First step is to create a new inherited scene from our Trigger scene. You can do this by finding your Trigger scene in your file system, right clicking it and selecting New Inherited Scene.

Next, go ahead and rename the root node of the new scene to Lever. The scene tree should now look like this:

Next select the AnimatedSprite2D node. In the inspector create a new sprite frames. Import the Lever sprite sheet from the Gates and Switches folder in your VGDC Assets folder.

We can keep the name default for the animation. Additionally, lets disable autoplay and looping, and set the animation speed to 15.

Next select the CollisionShape2D node in the scene tree. In the inspector create a new rectangle collision shape. We want the new shape to roughly match the size of the lever.

Creating the Lever Script

Of course the lever doesn't do anything yet. Let's add a new script to the Lever node. First, disconnect the Trigger script and create a new one inheriting our Trigger class.

In the lever script itself we need to do a couple things. First, let's add a class_name called Lever at the top of the script (above extends Trigger). Second, we will create a custom activate signal (to let any connected gates when to open), and finally a new variable called active, which we will use to make sure the lever can't be repeatedly pulled.

class_name Lever
extends Trigger

signal activate
var active = false

Finally, let's override the on_enter function from the Trigger class, having it emit our activate signal, play our lever's animation, set our active variable to true, and finally call the base on_enter function from our Trigger class. We also want to have a statement at the top of the function that will return if the active variable is true, making sure the lever is only activated once:

func on_enter(player):
	if active: return
	activate.emit()
	sprite.play()
	active = true
	super.on_enter(player)

Checkpoint! At this point you can place your lever into your level and see if it flips when the player collides with it!

Creating the Gate Scene

Now, our levers still have nothing to connect to. To remedy this let's make a gate. Create a new scene with a StaticBody2D node as a root node. Let's name the root node Gate. Next, create two children nodes for the Gate: an AnimatedSprite2D and CollisionShape2D node. The scene tree should now look like this:

Select the Gate node. In the inspector navigate to the Collision section and set the Gate's layer to be world and no masks. This will make the gate a solid surface like the floors or walls.

Next, select the AnimatedSprite2D node and create a new sprite frames. Import the gate open sprite from the Gates and Switches folder in your VGDC Assets folder.

Creating the Gate Script

Now on to the gate's functionality. Go ahead and add a new script to the Gate node. In it we want to create a new variable called lever (with the type of Lever, referencing the class we made earlier) as well as referencing our AnimatedSprite2D and CollisionShape2D nodes. You can add these nodes into your script by dragging them from the scene tree, holding control, then dropping them in the script. Let's rename their variables to sprite and collision_shape respectively.

@export var lever : Lever
@onready var sprite = $AnimatedSprite2D
@onready var collision_shape = $CollisionShape2D

Next, we want to create a function to open the gate and disable the collision when it recieves the activate signal from its lever. We can use a function like this to achieve it:

func open():
	sprite.play()
	collision_shape.disabled = true

Finally, let's connect the lever's activate signal to the gate. Let's call the ready function of our gate, and in it connect the lever's activate signal to our open function, like so:

func _ready():
	lever.activate.connect(open)

Connecting the Two

Our final step is to actually connect the levers and gates together! First place one of each into your world scene. Then, select the gate in the scene tree. You will see the Lever variable show up in the inspector. Next, drag the lever you want to connect to from the scene tree into the empty lever variable in the inspector.

And you're done! Go ahead and test if your gate opens properly, then move on to another module!
