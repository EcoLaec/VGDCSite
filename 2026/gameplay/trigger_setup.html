Trigger Setup

On this page we will create the functionality for the player to interact with non-hazardous objects (what I call triggers). In later modules we will use this for adding things like gems, health pickups and checkpoints.

Creating the Trigger Physics Layer

Before we can create any functionality for hazards, we need to let Godot know we want to track a new physics layer. Navigate to project settings, then find the Layer names -> 2D Physics option. Name Layer 4 Trigger.

Setting up the Player

Next, we need to add a way for the player to detect triggers. We can do this by going to our player scene, right-clicking our player node and adding a new Area2D node. Let's name the new node TriggerCollider. Then, right-click the new node and add a CollisionShape2D node.

Right now, however, our collider is still set to default layers and masks. For this specific collider we actually want to set it's mask to trigger and have no layer. This is because we want the actual object to be able to run its own script instead of the player's. We can set the collision layers in the inspector while having the TriggerCollider selected.

Next, select the new collision shape, go to the inspector and add a new rectange shape. Let's resize it to fit the player closer.

I generally make the trigger collider shape larger than the player's collider. This is to make collecting and interacting with positive objects easier. This makes the game feel more rewarding and "fair".

Of course, just setting up the collision doesn't do anything on its own. Next we want to connect our TriggerCollider's area_entered and body_entered signals to our player script. You can do this by selecting the TriggerCollider in the scene tree, then opening the signals tab in the inspector. Make both signals connect to a new function called _on_trigger_entered.

Note that we want both signals to connect to the same function. You can use the Pick button when connecting a signal to attach it to an existing function.

Next, let's repeat the process for the area_exited and body_exited signals, connecting them to a _on_trigger_exited function. After that, your signals tab should look like this:

Finally, lets add some code to those functions we connected the signals to. For each, we want to call a function (on_enter and on_exit respectively) of the trigger the player collided with, passing the player itself as an argument. This will allow us to have triggers effect the player. 

func _on_trigger_entered(trigger):
	trigger.on_enter(self)

Creating the Trigger Scene

Now, we will be using these triggers for many different modules with different functionality. That being said we do have some things that all of our triggers will need. We can reduce our need to repeat code and scenes by creating a base scene that our later modules can use! To get started, create a new scene with an Area2D as the root node. Setting up the Trigger Class

Our final piece of this puzzle is creating a new class that stores all the shared functionality of all triggers.

A class is kind of like a category. Any script can be made into a class, and other scripts can inherit from a class.

To get started, create a new script for our Trigger node. Have it inherit Area2D.

At the top of the script (above extends Area2D), add class_name Trigger. The keyword class_name tells Godot you want this to be a new class, with the name following immediately after. Next, lets make the function we called in the player script: on_enter, adding the player as an argument. For now, just set it to pass.

class_name Trigger
extends Area2D

func on_enter(_player):
	pass

You may notice the underscore before player in the function we made. When you add an underscore before a function's argument it lets Godot know you aren't planning on using it yet. If we didn't do this Godot would throw a warning letting us know we made a variable but never used it.

Finally, we want to be able to reference our CollisionShape2D and AnimatedSprite2D from our scene. You can drag them from the scene tree, hold control and drop them into the script. Let's change the variable names to sprite and collision_shape. Your script should now look like this:

class_name Trigger
extends Area2D

@onready var sprite = $AnimatedSprite2D
@onready var collision_shape = $CollisionShape2D

func on_enter(_player):
	pass
