Exploring the Editor

This page will get you more aquainted the Godot's editor, the place where we will be spending most of our time, as well as explore some important concepts of Godot's engine.

First Glance

Upon creating a new project, you will be welcomed by Godot's editor.

Your editor may look slightly different depending on your monitor's dimensions.

Godot's editor is made up of several parts, so let's go through them one by one.

Menus, Screens and Playtesting

At the very top of the editor there is the aptly named Top Bar.

The top bar is made up of three components: Menus, Screens and Playtesting Buttons.

  1. Menus

    The menus (the far left side of the top bar) are where we can find advanced options for our project (similar to most software used nowadays like Google Docs or Word.) In this course we will primarily be using the project tab to edit our project settings.

  2. Screens

    In the center of the top bar we can find our screens: 2D, 3D, Scripts and AssetLib. These change what you see in the viewport below. For this project we will be working with the 2D screen when we want to see the visuals of our project and the script screen while we program our scripts. For now select the 2D screen.

  3. Playtest

    This section on the far right of the top bar is where we will run our project when we wish to test our work. The only button we will be using for this course are the play button (and perhaps the stop button if things go very wrong.)

Scene Tree

Located on the left side of the editor, just below the menus, the Scene Tree is one of the most important parts of the editor.

The scene tree shows what nodes are part of the current scene and the relationships bewteen the nodes. If that confuses you that is alright, as these terms are specific to Godot. Let's break them down and put them into English.

  1. Nodes

    Nodes are components that add functionality. You can think of them like ingredients for a recipe or parts of a car. Some example of nodes are CharacterBody (for adding physics and collisions to something), AudioStreamPlayers (for making sounds) and Cameras (for controlling what the player sees.)

  2. Scenes

    Scenes are collections of nodes. They can be saved and are reusable. You can think of scenes as the actual objects of your game, like the player, enemies, the level and some more abstract things like the timer or pause menu.

  3. Relatedness

    Nodes are connected to eachother through relationships (also known as inheritance). A node can be a child of another node (this appears in the scene tree as the child node being indented slightly and having a line connecting it to its parent), which means it inherits from its parent node. Inheritance is an integral part of modern programming, but for our project, the most important part of it is that a child node will move with its parent node (it inherits it's position.)

If you are interested in knowing more about Godot's structure, you can check out their documentation Here

Viewport

The largest section of the editor, located in the center, is called the Viewport.

If your editor does not look right, make sure you have the 2D screen selected on the top bar

The viewport is one part level editor, one part code editor. In it we are able to preview the layout of our scenes and edit our scripts (depending on the screen selected on the top bar.) We are also able to quickly switch between our active scenes using the tabs at the top of the viewport.

File System

Located beneath the scene tree, the File System is where all of your saved assets can be seen.

The file system is functionally a view into your project's folder. In it you can store sprites, sounds, scenes, scripts or any other asset you use in your game (they don't even need to start with s.) It is very important to maintain organization in your files, especially as a project grows. You can right click in the file system dock to create a new folder.

Inspector

Located to the far right of the editor, the Inspector allows you to see and edit the details of nodes.

The inspector provides a detailed view of the node you currently have selected in the scene tree. It can look very different depending on the node, but almost every node requires some work in the inspector to have functionality. You can think of the inspector as the options page for nodes.

Bottom Panel

Located beneath the viewport, the Bottom Panel is the default location for additonal functionality in the editor.

The bottom panel is the home of our debugging as well as any additional functionality specific assets need. For example, when we begin working with animations, a new tab will appear on the bottom panel to allow us edit our animations.

Review

There is definitely a lot happening in the editor, and there was a lot skimmed over, but for the scope of this project the most important thing is you know where each part can be found, as well as these basic ideas:

  • Top Bar

    The top bar is where you can find project settings, the 2D and Script screen buttons to switch between visual and script views and the run button to test you game.

  • Scene Tree

    The scene tree is where we add and manage nodes to the selected scene.

  • Viewport

    The viewport is our workspace where we preview our scenes and program our code.

  • File System

    The file system is a view into our project's folder and where we save our assets.

  • Inspector

    The inspector is where we edit the details of the selected node.

  • Bottom Panel

    The bottom panel is home to the debugger and where we edit specific assets.

Once you are comfortable with the editor's layout, go ahead and move on to our next step: creating the player scene.
