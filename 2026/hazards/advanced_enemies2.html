Advanced Enemies (Part 2)

In the previous part we created the agile Neura. In this one we will create the Dilo.

Creating the Dilo Scene

Just like all the other enemies, the Dilo starts by creating a new inherited scene from our enemy scene.

Next, rename the Enemy node to Dilo and select the AnimatedSprite2D and create a new sprite frames in the inspector. Then let's import all of the Dilo spritesheets: move, death, idle, spit and recover, as well as the acid sprite. You can find them in the Advanced Enemies folder in your VGDC Assets folder.

If you are not taking this course in person, you can download the assets Here

Note that we want both the spit and recover animations to not be looped and set to the same speed (I chose 12).

Next, select the CollisionShape2D node and create a new rectangle shape in the inspector. Let's resize it to fit the sprite.

Now for the new nodes. We will want three of them. First a Timer node named StateTimer and a new Area2D node, which I will name Vision. Finally, create a CollisionShape2D node as a child of the Vision node.

Next, select the StateTimer node. In the inspector we want to set it to both be Autostart and One-shot.

Now to handle the Vision node. Select it and navigate to the inspector. We want to set it's layers to be blank and its masks to be just our player layer.

Finally, select the CollisionShape2D node under the Vision node. Just like the Neura we will create a new CircleShape2D in the inspector and size it in the viewport.

Making the Dilo Script

Just like the Neura, we will be making a new script for the Dilo. Go ahead and disconnect the default Enemy script from the Dilo node, then attach a new one. Let's name the new script dilo and make it inherit from the Enemy class.

To start, we will want five new variables: an export variable called bullet, which will point to our future acid bullet scene, rng, a new RandomNumberGenerator which will will use to randomize the Dilo's actions, a variable to check if the Dilo recently spit acid, target, which will be used to track the player, a current_speed variable and a reference to our StateTimer node.

extends Enemy

@export var bullet : PackedScene
var rng = RandomNumberGenerator.new()
var just_spit = false
var target : Node2D
var current_speed = speed
@onready var state_timer = $StateTimer

Note that if you have completed the Aquatic Enemies module, you do not need the current_speed variable.

Next, we will want to prep our rng variable in the _ready function, randomizing it when the game starts:

func _ready():
	rng.randomize()

Additionally, we want to override our handle_direction function we are using for our enemies. Specifically, we want to call the original function using super, then manually set the sprite's direction based on whether or not the Dilo can see the player.

func handle_direction():
	super.handle_direction()
	if not target:
		sprite.flip_h = false if direction.x == 1.0 else true
	else:
		sprite.flip_h = false if global_position.x - target.global_position.x < 0.0 else true

Next, just like the Neura, we want to set up the Vision node. In fact, it is the exact same!

func _on_player_spotted(body):
	target = body

func _on_player_lost(_body):
	target = null

Finally, for the most complex part we want to connect the StateTimer's timeout signal to the script.

Within this new function we will handle all of the Dilo's complex behavior. Just like the Neura I won't be describing each part, but I added comments to try and grant some insight.

func _on_state_timer_timeout():
	# Restarting the State Timer
	state_timer.start(rng.randf_range(0.5,1.25))
	if sprite.animation == "death": return
	
	# Handling Different States
	if sprite.animation == "idle":
		if rng.randf() < 0.66 and target and not just_spit:
			# Spitting
			if rng.randf() < 0.5: velocity.y = -300.0
			sprite.play("spit")
			just_spit = true
			await sprite.animation_finished
			if sprite.animation == "death": return
			sprite.play("recover")
			# Spawn Acid
			var b = bullet.instantiate()
			get_parent().add_child(b)
			b.global_position = global_position
			b.velocity.x *= -1.0 if sprite.flip_h else 1.0
			await sprite.animation_finished
			if sprite.animation == "death": return
			sprite.play("idle")
			return
		# Wandering
		if rng.randf() < 0.5: change_direction()
		current_speed = speed
		sprite.play("move")
		just_spit = false
	else:
		# Idling
		current_speed = 0.0
		sprite.play("idle")

Creating the Acid Bullet

Finally, we need to create the projectile the Dilo will actually spit! To get started create a new scene with an Area2D as its root node. Let's name it AcidBullet. Next, create three nodes as children of the AcidBullet: a Sprite2D node (not AnimatedSprite2D!), a CollisionShape2D node, and a Timer node named LifeTimer.

To start, select the AcidBullet node and in the inspector set it's collision to be on the hazards layer and no masks.

Go ahead and select the Sprite2D node, dragging the Acid sprite from your file system into the texture property in the inspector.

Next, select the LifeTimer node and in the inspector set it to be Autostart, One-shot and its wait time to be 5 seconds.

Finally for the scene tree, select the CollisionShape2D node and create a new CircleShape2D in the inspector, shrinking it to be slightly smaller than the acid sprite.

Now onto the script! Go ahead and make a new script on the AcidBullet node. In it we will manually recreate some of the physics in our player and enemy scripts, with some differences. Let's start with the variables:

@export var terminal_velocity = 250.0
@export var physics_gravity = 250.0
@export var friction = 25.0
var velocity = Vector2(100.0,-50.0)

Most of this is just like our player, except we needed to change the name of gravity to physics_gravity (since Area2Ds have a gravity variable that does something different already), and our velocity variable is set to an initial value. Though we can continue reusing our code by overriding our _physics_process function.

func _physics_process(delta):
	velocity.y = move_toward(velocity.y, terminal_velocity, physics_gravity * delta)
	velocity.x = move_toward(velocity.x, 0.0, friction * delta)
	position += velocity * delta

Our final step for our AcidBullet is to connect the timeout signal from our LifeTimer to the script, deleting the projectile when its time runs out (so we don't fill the world with infinite projectiles).

func _on_life_timer_timeout():
	queue_free()

Finally, to make sure the Dilo knows what it needs to be spawning, go ahead and save your AcidBullet scene. Next, in the Dilo scene select the Dilo node and drop your new AcidBullet scene from the file system into the Bullet property in the inspector.

And you're done! Go ahead and add some Dilo's to your level, check how they work, then move on to another module!
