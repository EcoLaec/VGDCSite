Aquatic Enemies

IMPORTANT! This module requires both Simple Enemy modules to be completed first!

In this module we build off of our basic enemy to create some more enemies for underwater sections: the Anomalo and Nauty!

Note that each of these enemies can be added individually. If you don't want one, just skip it!

Creating the Anomalo

The anomalo will be a simple enemy that floats (or flies) without falling. To create it, we will repeat the same steps we used for the trike in the Simple Enemy (Part 2) module, starting by creating a new inherited scene from our enemy scene.

Next, rename the Enemy node to Anomalo, select the AnimatedSprite2D node and create a new sprite frames in the inspector. Then let's import the Anomalo move and death spritesheets. You can find them in the Aquatic Enemies folder in your VGDC Assets folder.

If you are not taking this course in person, you can download the assets Here

Next, select the CollisionShape2D node and create a new rectangle shape in the inspector. Let's resize it to fit the sprite.

Finally, to make our Anomalo swim we actually don't need to make any changes in our enemy script! Instead, select the Anomalo node and go to the inspector. Let's set its speed to 50.0 and its gravity to 0.0.

And that is the Anomalo! You may note that there is nothing actually requiring it to be underwater. This is by design, as there is no need to overcomplicate things. If you want a flying Anomalo in the sky, I am not going to stop you. Go ahead and save the scene and add some swimming enemies to your level.

Creating the Nauty

This section requires the Ankylo section of the Intermediate Enemies module.

The Nauty will be quite similar to the Anomalo, except it will be moving in a rythmic pulsing motion instead of a constant speed, as well as being stunned instead of killed when jumped on. To get started go to the enemy script. We will be adding two new variables: current_speed and friction.

@export var speed = 25.0
@export var gravity = 980.0
@export var friction = 0.0
@export var invincible = false
@onready var sprite = $AnimatedSprite2D
@onready var collision_shape = $CollisionShape2D
var direction = Vector2.RIGHT
var is_moving = true
var current_speed = speed

Next, go to the handle_movement function. In it we want to update our current_speed using our friction as well as use current_speed instead of speed when figuring out our velocity.x.

func handle_movement(delta):
	current_speed = move_toward(current_speed, 0.0, friction * delta)
	velocity.x = direction.x * current_speed

Up to this point however, we have no ways to reset the speed, which would cause the enemy to just slow down to a halt. To fix this lets select the AnimatedSprite2D node in the enemy scene, then go to the signals tab in the inspector. Let's connect the animation_looped signal to a function named _on_animation_looped.

Next let's edit the new _on_animation_looped function in our enemy script to reset our current_speed like so:

func _on_animation_looped():
	current_speed = speed

Though right now we would still have a problem: when the Nauty is hurt its speed gets set to 0. To make sure it begins moving again let's reset the current_speed for invincible enemies when they start moving again in the die function.

func die():
	is_moving = false
	if not invincible: call_deferred("disable_collision")
	sprite.stop()
	sprite.play("death")
	await sprite.animation_finished
	if invincible:
		is_moving = true
		sprite.play("move")
		current_speed = speed
	else:
		queue_free()

Finally we can finally make the Nauty itself! In the filesystem right click the enemy scene and create a new inherited scene. Change the name of the root node to Nauty, then repeat the same process we did for the Anomalo and Trike to add the Nauty's sprites, animations and collision shape (you can find the sprites in the Aquatic Enemies folder of your VGDC Assets folder.)

For the Nauty's animations, however, we will want to modify some of the frame durations, specifically frame 3 of the death animation (which I set to 15.0) and frame 0 of the move animation (which I set to 6.0).
