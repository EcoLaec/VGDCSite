Time Limit

IMPORTANT! This module requires the Hazard Setup and UI modules to be completed first!

On this page we will make the game a more stressful by adding a time limit!

Creating the Timer

In order to have a timer we need, well, a timer. First go to your world scene, select the root node and add a new timer node as a child. Let's name it LevelTimer.

In its inspector, check one-shot so that it is active, and keep all the other settings as default.

Editing the World Script

Now let's set up the ability to access our level timer in the world script (you can add a script to your world scene if you don't already have one). First, let's add a new export variable called level_time_seconds, which will be the amount of time (in seconds) that the timer runs for. I'll set my default to 300, or 5 minutes:

@export var level_time_seconds = 300

Next, let's call the world script's _ready function, and in it we can start the LevelTimer using the level_time_seconds using the following code:

func _ready():
	$LevelTimer.start(level_time_seconds)

Finally we want to have the player respawn when the time runs out. We can do this by first selecting the LevelTimer node and in the inspector go to the signals tab and connect the timeout signal to the world script.

To finish up, go to the new _on_level_timer_timeout function in the world script. We want to first call the player's respawn function, then restart the timer like so:

func _on_level_timer_timeout():
	$Player.respawn()
	$LevelTimer.start(level_time_seconds)

Adding the UI Element

Now, while we have all the functionality working, we still cannot see how much time we have left. In order to fix this lets go to our UI scene. Select the VBoxContainer under the Center node, then create a new Label node as a child of the VBoxContainer. Let's name this new label Time.

Select the Time node, right click it and select Access as Unique Name. We do this to be able to connect it in our script even if we later decide to move it in the scene tree. Next open your UI script and In the script's _process function add the following line of code:

func _process(_delta):
	%Time.text = str(int(level_timer.time_left / 60)).pad_zeros(2) + ":" + str(int(level_timer.time_left) % 60).pad_zeros(2)

The line we added simply turns the time left on our LevelTimer into minutes and seconds, so we can display it in an easy to understand way. If we just put in the time left we would have so many decimal places!

You will see some errors. This is because we haven't set up the level_timer variable in our UI script! We can fix this by adding an export variable in the UI script called level_timer and with the Timer datatype, like so:

@export var level_timer : Timer

To actually connect the two, go back to your world scene and select your UI node. Drag the LevelTimer from the scene tree into the Level Timer variable in the inspector.

And You are done! Go ahead and set your level(s) time limits and try it out! Once you are done go ahead and try another module.
