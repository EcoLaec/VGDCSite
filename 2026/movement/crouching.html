Crouching

Simply put, we can add the ability for the player crouch down, reducing the size of their collision to get into tight spaces.

Getting the New Assets

Before we can start adding the ability to crouch, we need the new sprites! Go ahead and open up the Crouching folder in your VGDC Assets folder, find the appropriate files and drag it into your file system.

If you are not taking this course in person, you can download the assets Here

Now we need to add a new animation to our player's sprite frames. Navigate to your player scene's AnimatedSprite2D then it's sprite frame. We will create two new animations named crouch_idle and crouch_walk. Go ahead and import the new files as frames.

I set my crouch_idle animation speed to 4 and my crouch_walk speed to 8 (80% of the regular speeds). You can go ahead and change it to what you want, but I do recommend slower than your regular idle/walking.

Adding the Input

Next we must add a new input action for crouching. Go to your project settings then the input map tab. Create a new action called Down and add a key (I will use S and the down arrow as my down keys.)

Setting up the Player Scene

Next we will need to add a couple things to our player's scene tree. First, add a new CollisionShape2D node to our player, rename it to CrouchCollision and move it beneath our original CollisionShape2D. This will be the collision we use while crouching. We also want to add a new Area2D node named HeadCheck, as well as another CollisionShape2D as a child of HeadCheck, naming it HeadCollision. This will be used to prevent the player from getting stuck in the ceiling.

Next let's add the actual shapes to our new CollisionShape2Ds. Select our CrouchCollision node, and in the inspector add a new RectangleShape to it. Edit the shape in the Viewport to the size you want. I made mine 4 pixels shorter than the normal collider. Then select HeadCollision and add another RectangleShape, editing it to take up the remaining space between the CrouchCollision shape and default collision, like so:

Going into your player's AnimatedSprite2D and selecting the crouch_idle animation can make sizing the box much easier.

Now we need to make sure that our HeadCheck is connecting properly with our script. We can start this by first selecting HeadCheck and in the inspector changing its collision layers to be empty and keeping its masks on world:

Finally, still in HeadCheck's inspector, go to the signals tab, connecting both the body_entered and body_exited signals to our script by double clicking them (you can keep the default name and settings for both.)

Editing the Player Script

To get started in the player script we will need to add 6 new variables! First, we will add crouch_speed to our player stats, which I set to 80.0. Next we will need to get access to our default CollisionShape2D, CrouchCollision and HeadCheck in our script. We can get them by starting to drag them from our scene tree, holding control and then dropping them into our script (keeping the default names). Finally, we will add two more reference variables, those being crouching and head_blocked, both started as false:

# Player Stats
@export var speed = 100.0
@export var acceleration = 800.0
@export var friction = 1200.0
@export var jump_velocity = -300.0
@export var gravity = 980.0
@export var terminal_velocity = 500.0
@export var crouch_speed = 80.0

# Reference Variables
@onready var sprite = $AnimatedSprite2D
@onready var head_check: Area2D = $HeadCheck
@onready var collision_shape_2d: CollisionShape2D = $CollisionShape2D
@onready var crouch_collision: CollisionShape2D = $CrouchCollision
var crouching = false
var head_blocked = false

Next up let's add a new method into our _physics_process funtion, that being handle_crouching(). No need for any arguments, as this function will only manage starting and ending our crouching state.

func _physics_process(delta):
	var input_axis = Input.get_axis("Left","Right")
	handle_crouching()
	apply_gravity(delta)
	handle_jump()
	apply_acceleration(input_axis,delta)
	apply_friction(input_axis,delta)
	update_animations(input_axis)
	move_and_slide()

Of course we will get an error, so lets quickly make our handle_crouching function. In the new function we want to test if the player is holding the down input and is currently on the ground. If so, we will enable the crouch_collider, disable our default collider and set crouching to true. If the condition isn't met, we want to check if the player's head is blocked, and if not, reverse the changes we made (we check head_blocked so the player can't uncrouch when they would be inside of a wall).

func handle_crouching():
	if is_on_floor() and Input.is_action_pressed("Down"):
		crouching = true
		crouch_collision.disabled = false
		collision_shape_2d.disabled = true
	elif head_blocked == false:
		crouching = false
		collision_shape_2d.disabled = false
		crouch_collision.disabled = true

Right now, however, we aren't actually updating if the player's head is blocked! To fix this we will edit the functions we added earlier through the HeadCheck's signals. We will simply set head_blocked to true when HeadCheck enters a body, and set head_blocked to false when it exits one, like so:

func _on_head_check_body_entered(_body):
	head_blocked = true

func _on_head_check_body_exited(_body):
	head_blocked = false

On a related note, we also want to prevent the player from jumping if their head is blocked. We do this simply by returning out of the handle_jump function if head_blocked is true:

func handle_jump():
	# Blocked Jumping
	if head_blocked == true: return
	
	# Jumping
	if is_on_floor() and Input.is_action_just_pressed("Jump"):
		velocity.y = jump_velocity

Now we are ready to actually apply some actual functionality! First lets apply our crouch_speed while crouching. We do this by creating a new current_speed variable in our apply_acceleration function, setting its value based on an if then statement:

func apply_acceleration(input_axis,delta):
	# Set Speed
	var current_speed
	if crouching:
		current_speed = crouch_speed
	else:
		current_speed = speed
		
	if input_axis:
		velocity.x = move_toward(velocity.x, current_speed * input_axis, acceleration * delta)

Note that speed was replaced with current_speed in the move_toward method.

Finally, let's make sure our crouching animations play by editing our update_animations function. Specifically we want to add a condition to play our crouch_walk and crouch_idle animations instead of the default ones while crouching, like so:

func update_animations(input_axis):
	if input_axis: sprite.flip_h = input_axis < 0
	
	# Grounded Animations
	if input_axis:
		if crouching: 
			sprite.play("crouch_walk")
		else:
			sprite.play("walk")
	else:
		if crouching:
			sprite.play("crouch_idle")
		else:
			sprite.play("idle")
	
	# Aerial Animations
	if not is_on_floor():
		if velocity.y <= 0:
			sprite.play("jump")
		else:
			sprite.play("fall")

And we are done! Go ahead and test out your new ability, mess around with the speed and once you are happy with it, move on to another module!

Compatibility

Crouching conflicts with 3 other modules:

  1. Hazard Setup
  2. Sprinting and Swimming

If you have any of these modules please follow the instructions for that module:

Hazard Setup

At the moment, crouching only effects the player's collision with the level itself, not hazards. To fix this let's start by adding a new CollisionShape2D node to our HazardCollider in the player scene. Let's name it HazardCrouchCollider.

Next, select the new HazardCrouchCollider and in the inspector add a new RectangleShape to it. Edit its size to be proportional with your CrouchCollider shape. Mine looks like so:

Just like the default hazard collision shape, it is good to make the CrouchHazardCollision shape slightly smaller than your CrouchCollision shape in order to slightly improve the feel of the game.

Now we need to edit our player script. First let's add references to our two HazardCollider collision shapes in our reference variables, like so:

# Reference Variables
@onready var sprite = $AnimatedSprite2D
@onready var head_check: Area2D = $HeadCheck
@onready var collision_shape_2d: CollisionShape2D = $CollisionShape2D
@onready var crouch_collision: CollisionShape2D = $CrouchCollision
@onready var hazard_collision_shape_2d: CollisionShape2D = $HazardCollider/CollisionShape2D
@onready var hazard_crouch_collision: CollisionShape2D = $HazardCollider/HazardCrouchCollision

Note that I added hazard before the variable name for our HazardCollider's collision shape. This is because we already have a collision_shape_2d variable from before!

Finally, we will add code to disable and enable our two hazard collision shapes in our handle_crouching function, mirroring the default collision:

func handle_crouching():
	if is_on_floor() and Input.is_action_pressed("Down"):
		crouching = true
		crouch_collision.disabled = false
		collision_shape_2d.disabled = true
		hazard_crouch_collision.disabled = false
		hazard_collision_shape_2d.disabled = true
	elif head_blocked == false:
		crouching = false
		collision_shape_2d.disabled = false
		crouch_collision.disabled = true
		hazard_collision_shape_2d.disabled = false
		hazard_crouch_collision.disabled = true

Sprinting and Swimming

Since each of these modules modify the player's max speed, we need to set the order of priority, with crouching overriding sprinting and swimming overriding both. We can do this using elif (else if) statements like so:

func apply_acceleration(input_axis,delta):
	sprinting = Input.is_action_pressed("Sprint")
	
	# Set Speed
	var current_speed
	if in_water:
		current_speed = swim_speed
	elif crouching:
		current_speed = crouch_speed
	elif sprinting:
		current_speed = sprint_speed
	else:
		current_speed = speed
	
	# Set Acceleration
	var current_acceleration
	if in_water:
		current_acceleration =swim_acceleration
	else:
		current_acceleration = acceleration
	
	# Deacceleration after Dash
	if abs(velocity.x) > current_speed: velocity.x = move_toward(velocity.x, current_speed * input_axis, dash_acceleration * delta)
	
	if input_axis:
		velocity.x = move_toward(velocity.x, current_speed * input_axis, current_acceleration * delta)

Swimming Continued

In addition to the previous fix, we want swimming to actually prevent crouching entirely. We can do this by adding a condition to check if the player is in water in our handle_crouching function:

func handle_crouching():
	if is_on_floor() and Input.is_action_pressed("Down") and not in_water:
		crouching = true
		crouch_collision.disabled = false
		collision_shape_2d.disabled = true
	elif head_blocked == false:
		crouching = false
		collision_shape_2d.disabled = false
		crouch_collision.disabled = true
