Swimming (Part 2)

In this module we will add the ability for our player to swim in water!

Getting the New Assets

Before adding any functionality lets get the new sprites. Go ahead and open up the Swimming folder in your VGDC Assets folder, find the appropriate files and drag them into your file system.

If you are not taking this course in person, you can download the assets Here

Now we need to add a couple new animations to our player's sprite frames. Navigate to your player scene's AnimatedSprite2D node, then it's sprite frame. We will create three new animations: swim, swim_idle and swim_up, importing the appropriate files as sprite frames.

I set my swim animation speed to 5, my swim_idle speed to 4 and swim_up to 7. You can go ahead and change it to what you want, I just did what I thought looked good.

Checking for Water

Next we want a way for the player to detect when they are in water. In the player scene create a new Area2D node and name it WaterCheck, then create a new CollisionShape2D node as a child of it.

Now select the new CollisionShape2D and in the inspector create a new RectangeShape. Edit it in the viewport to the proper shape and size.

I made my WaterCheck shape cover only the top portion of my character. I did this because I think it looks nicer when the player doesn't start swimming immediately when touching a single pixel of water, though note this does make it harder to leave the water.

Next, select WaterCheck and in the inspector, navigate to the collision section. Disable its layers and set its masks to Water only.

Finally, we want to connect the WaterCheck node to our player script. While still in the inspector for WaterCheck, go to the signals tab. We want to connect both the body_entered and body_exited signals (you can keep the default function names).

Editing the Player Script

We have a lot going on here, so stay with me. First we will want to add 7 new variables to our player: swim_speed, swim_acceleration, swim_friction, swim_jump_velocity, swim_gravity, swim_terminal_velocity and in_water.

# Player Stats
@export var speed = 100.0
@export var acceleration = 800.0
@export var friction = 1200.0
@export var jump_velocity = -300.0
@export var gravity = 980.0
@export var terminal_velocity = 500.0
@export var swim_speed = 65.0
@export var swim_acceleration = 400.0
@export var swim_friction = 600.0
@export var swim_jump_velocity = -150.0
@export var swim_gravity = 300.0
@export var swim_terminal_velocity = 75.0

# Reference Variables
@onready var sprite = $AnimatedSprite2D
var in_water = false

We are pretty much adding an alternative value for every movement stat we have. I chose these values based on what felt good to me, but feel free to mess around with them!

Now let's start applying our swim stats one function at a time, starting in apply_gravity. We will want to be able to switch between our two gravities and terminal velocities based on if we are in water, like so:

func apply_gravity(delta):
	# Set Terminal Velocity
	var current_terminal_velocity
	if in_water:
		current_terminal_velocity = swim_terminal_velocity
	else:
		current_terminal_velocity = terminal_velocity
	
	# Set Gravity
	var current_gravity
	if in_water:
		current_gravity = swim_gravity
	else:
		current_gravity = gravity
	
	if not is_on_floor():
		velocity.y += current_gravity * delta
		velocity.y = min(velocity.y,current_terminal_velocity)

Next, handle_jump. Unlike gravity, we aren't just changing between two variables, but actually need to modify our jumping statement. Specifically we will need to pull our is_on_floor check and put it inside of the statement, returning out of it if we are in the air and not in water. This will allow the player to jump (aka swim) in water without needing to touch the floor!

func handle_jump():
	# Set Jump Velocity
	var current_jump_velocity
	if in_water:
		current_jump_velocity = swim_jump_velocity
	else:
		current_jump_velocity = jump_velocity
	
	# Jumping
	if Input.is_action_just_pressed("Jump"):
		# Swimming
		if not is_on_floor() and not in_water:
			return
		
		velocity.y = current_jump_velocity

Now apply_acceleration, where we modify our speed and acceleration:

func apply_acceleration(input_axis,delta):
	# Set Speed
	var current_speed
	if in_water:
		current_speed = swim_speed
	else:
		current_speed = speed
	
	# Set Acceleration
	var current_acceleration
	if in_water:
		current_acceleration = swim_acceleration
	else:
		current_acceleration = acceleration
	
	if input_axis:
		velocity.x = move_toward(velocity.x, current_speed * input_axis, current_acceleration * delta)

And finally apply_friction, with only friction to deal with.

func apply_friction(input_axis,delta):
	# Set Friction
	var current_friction
	if in_water:
		current_friction = swim_friction
	else:
		current_friction = friction
	
	if not input_axis:
		velocity.x = move_toward(velocity.x, 0, current_friction * delta)

Note that each of these functions have the current versions of each variable being applied at the bottom!

As you can tell, there is a lot of changes in order to swim, but we aren't done yet. Let's make sure our animations are properly working! In the update_animations function we will add a section above Grounded Animations for our swimming animations. We want swim_up to play while moving upwards, swim while moving, and idle otherwise. We also want our swimming animations to override the regular ones, so we will return out of the function afterwards:

func update_animations(input_axis):
	# Update Sprite Direction
	if input_axis: sprite.flip_h = input_axis < 0
	
	# Swimming Animations
	if in_water:
		if velocity.y < 0.0:
			sprite.play("swim_up")
		elif input_axis:
			sprite.play("swim")
		else:
			sprite.play("swim_idle")
		return
	
	# Grounded Animations
	if input_axis:
		sprite.play("walk")
	else:
		sprite.play("idle")
	
	# Aerial Animations
	if not is_on_floor():
		if velocity.y <= 0:
			sprite.play("jump")
		else:
			sprite.play("fall")

We are so close to being done! We just need to actually update the in_water variable we have been calling! We can do this easily by setting in_water to true in the _on_water_check_body_entered function and false in the _on_water_check_body_exited function (the two functions created when we connected the WaterCheck's signals earlier).

func _on_water_check_body_entered(_body):
	in_water = true

func _on_water_check_body_exited(_body):
	in_water = false

And you're done! Go ahead and try out swimming and move on to another module!

Compatibility

Swimming conflicts with 7 other modules:

  1. Aerial Mobility
  2. Crouching
  3. Double Jump
  4. Short Hop
  5. Sprinting
  6. Wall Jump
  7. Wall Cling

Due to the fact this module is already long enough (and has a lot of conflicts), you can go to each of these modules' pages to see how to fix the compatability issues.
