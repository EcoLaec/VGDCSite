Hurt Effects

IMPORTANT! This module requires the Hazard Setup module to be completed first!

At this point dying in the game can be a bit jarring, with an instant teleport with no delay or visual way of showing what happened. In this module we will explore adding some more visual feedback to respawning and taking damage.

Death Effects

Before we can start adding the death animation, we need the new sprites! Go ahead and open up the Hurt Effects folder in your VGDC Assets folder, find the appropriate files and drag it into your file system.

If you are not taking this course in person, you can download the assets Here

Now we need to add a new animation to our player's sprite frames. Navigate to your player scene's AnimatedSprite2D then it's sprite frame. We will create a new animation named death, using the corresponding death spritesheet.

Note that the death animation is set to not loop. If you don't do this it will break later!

Next, while we have the AnimatedSprite2D selected we will want to go in the inspector and under the process section set the mode property to Always.

Setting this property dictates how a node will interact with Godot's built-in pausing. In this case we want the death animation to play even when we freeze the rest of the game, and the Always process mode allows that. Though of course right now nothing is freezing, or even playing! To fix that go to your player's script and in the respawn function add the following code:

The following code is redundant if you have Health in your game. If you do, skip this code and move on to the next section instead!

func respawn():
	sprite.play("death")
	get_tree().paused = true
	await sprite.animation_finished
	get_tree().paused = false
	position = respawn_point

This code plays the death animation, pauses the rest of the game, then waits for the animation to end before un-pausing and respawning the player. Go ahead and test it out!

Damage Effects

This section requires both of the Health modules. If you do not have health in your game you can safely ignore this section.

Of course, a death animation is a bit much if you have health in your game. To add effects to that, however, things get a bit more complex. To get started go to your player scene's AnimatedSprite2D then it's sprite frame. We will create a new animation named hurt, using the corresponding hurt spritesheet.

Note that just like the death animation, the hurt animation is set to not loop!

Next, to use our new animation, let's return to our player script and in the take_damage function add the following code:

func take_damage(force_respawn : bool):
	if invincibility_timer.time_left == 0.0 or force_respawn:
		health -= 1
		if health <= 0 or force_respawn:
			sprite.play("death")
		else:
			sprite.play("hurt")
		get_tree().paused = true
		await sprite.animation_finished
		get_tree().paused = false
		if health <= 0:
			respawn()
			health = max_health
		if force_respawn: position = respawn_point

This code makes it that the proper animation plays, then the game pauses until the animation finishes, then continues on as normal. Though there is one final component of our damage system that could use some visual flair: the invincibility frames. To remedy this we want to create a new node that is child to our Player node. Specifically we will make an AnimationPlayer node.

An animation player is a special node that allows us to set smooth transitions between different things called Key Frames. Each key frame is a preset value we set for any given property (such as position, color, etc.). To get started with them let's select the new AnimationPlayer node. You will see a new editor pop up in the bottom bar.

There is a lot going on here, but we will take it one step at a time. To start, press the Animation button and create a new animation. Let's name it flicker.

Next make sure you have your flicker animation selecting in the drop down next to the Animation button. Next, on the right side of the animation editor we can set the animation length and if it loops. Let's set it to be 0.25 seconds long and loop.

Now it's time to set up our keyframes! Select your AnimatedSprite2D node while your animation editor is open (you may need to press the animation tab in the bottom bar). You will notice a bunch of small key icons next to properties in the inspector. These buttons are how we set key frames. Navigate to the visibility section and press the key icon next to the modulate property, leaving the following pop up as default to make a reset track.

The modulate property is a way to change the color and transparency of a node without needing to change its sprite.

You will see a white square pop up in the animation editor. This is your first key frame! You can select it to change its values and time in the animation. To add our next one move let's right click in an empty space on our animation track and select duplicate keys. Select the new key and in the value property, open the color selector and drag the alpha bar to the left, making the color completely transparent.

Next, still in the key frame inspector, set the time to be 0.125 (exactly half way through our animation).

Finally, to finish off our flicker animation we want to set its transition mode to be nearest. We can change it just under the animation duration from the blue wavy line (cubic) to the boxy line (nearest).

At this point the flicker animation is complete, and should look like so:

Now to actually use it! Back in the player script we will want to create a reference variable to our AnimationPlayer node.

@onready var animation_player = $AnimationPlayer

Next, go back to our take_damage function, adding a line to start our flicker animation after the scene is un-paused.

func take_damage(force_respawn : bool):
	if invincibility_timer.time_left == 0.0 or force_respawn:
		health -= 1
		if health <= 0 or force_respawn:
			sprite.play("death")
		else:
			sprite.play("hurt")
		get_tree().paused = true
		await sprite.animation_finished
		get_tree().paused = false
		invincibility_timer.start()
		animation_player.play("flicker")
		if health <= 0:
			respawn()
			health = max_health
		if force_respawn: position = respawn_point

And finally, to stop the flickering, let's reset our animations when the invincibility timer timeouts:

func _on_invincibility_timer_timeout():
	animation_player.play("RESET")
	$HazardCollider/CollisionShape2D.disabled = true
	$HazardCollider/CollisionShape2D.disabled = false

And you're done! Go ahead and check out the new effects, then move onto another module!

Compatibility

Hazard Sounds

Right now, the player's hurt sounds get paused when the rest of the scene is paused. To fix this select the HurtSound node and in the inspector set its process mode to Always.

And that's all you need to do!
