Level Transition

IMPORTANT! This module requires the Multiple Levels module to be completed first!

Right now changing levels is quite abrupt, with just an instant switch between each. We can make a simple animation to fade in and out of black to make that transition quite a bit smoother.

Setting Up the World Scene

To get started go to your world scene. In it create a new CanvasLayer node. Let's name it LevelTransition. Next, create two more nodes as children of the LevelTransition node: a ColorRect node and an AnimationPlayer node.

Right now our ColorRect is just a small square, but we will want it to cover the full screen. We can do this select the ColorRect node and at the top of the viewport select its anchors, which will be an circle icon with a + symbol in it. Select Full Rect.

Next for the color. While still having the ColorRect node selected, go to the inspector and select the Color property. A menu will pop up for you to select a color. I will have mine set to #181425.

Setting Up the Animations

Right now, however, our ColorRect would only block our view of the game! To get started fixing this issue we will need to set up some animations using our AnimationPlayer node. An animation player is a special node that allows us to set smooth transitions between different things called Key Frames. Each key frame is a preset value we set for any given property (such as position, color, etc.). To get started with them let's select the our AnimationPlayer node. You will see a new editor pop up in the bottom bar.

Don't worry, we will go step by step. To start, press the Animation button and create a new animation. Let's name it fade in.

Next make sure you have your fade in animation selecting in the drop down next to the Animation button. Next, on the right side of the animation editor we can set the animation length and if it loops. Let's set it to be 0.5 seconds long.

Now it's time to set up our keyframes! Select your ColorRect node while your animation editor is open (you may need to press the animation tab in the bottom bar). You will notice a bunch of small key icons next to properties in the inspector. These buttons are how we set key frames. Go ahead and press the key icon next to the color property, leaving the following pop up as default to make a reset track.

You will see a square pop up in the animation editor. This is your first key frame! You can select it to change its values and time in the animation. To add our next one move let's right click in an empty space on our animation track and select duplicate keys. Select the new key and in the value property, open the color selector and drag the alpha bar to the left, making the color completely transparent.

Next, still in the key frame inspector, set the time to be 0.5 (exactly at the end of the animation).

At this point the fade in animation is complete, and should look like so:

Connecting the Scripts

Finally we need to actually let the game know when to play the animations! We can start by opening our world script. In it we want to first add a reference variable to our AnimationPlayer.

@onready var animation_player = $LevelTransition/AnimationPlayer

Next, in the _ready function we want to play our fade in animation.

func _ready():
	animation_player.play("fade in")
	Global.level_complete.connect(complete_level)

And finally, when we complete a level we want to play the fade out animation, wait for it to complete, and then switch levels.

func complete_level():
	animation_player.play("fade out")
	await animation_player.animation_finished
	if not next_level is PackedScene: return
	get_tree().change_scene_to_packed(next_level)

And you're done! Go ahead and make sure everything is working, mess aruond with the animations, and move on to another module!
