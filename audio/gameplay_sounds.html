Gameplay Sounds

IMPORTANT! This module requires the Trigger Setup module to be completed first!

Continuing with sounds, at the moment our pickups, checkpoints and switches are completely silent and lifeless! If you have completed the previous two sound-based modules, I would challenge you to try and add sound effects to the triggers without going through this module, but if you haven't, or you run into any issues here's how to do it:

Sound Setup

First things first, navigate to your trigger scene. Right click the Trigger node and create a new AudioStreamPlayer node as a child. Rename it to TriggerSound.

Then, in the trigger script we will want to edit the on_enter function to play the sound when triggered, like so:

func on_enter(_player):
	$TriggerSound.play()

Pickup Sounds

This section requires the Collectables module. If you do not have collectables in your game you can safely ignore this section.

First we need to import our pickup sound(s) from the Gameplay Sounds folder in your VGDC Assets folder.

Now the pickup will play the sound when it is collected. However, if you test the game out now you may notice a problem: the sound gets cut off immediately! This is because we are deleting the pickup the moment the player touches it, which also deletes the audio player. To fix this we need to remove the queue_free() from the on_enter function in the Pickup script, then create a new function called disable_pickup with the following code:

func disable_pickup():
	$CollisionShape2D.disabled = true
	visible = false
	await $TriggerSound.finished

This function deactivates the pickup's collision and visuals, effectively removing it for the player, then waits for the sound to be done before deleting the pickup. In order to use this function lets add a call_deferred method in the on_enter function.

func on_enter(player):
	super.on_enter(player)
	call_deferred("disable_pickup")

Note that the call_deferred method simply waits to call a method until after the current physics frame is complete. We do this as deactivating collision within a physics frame can break things pretty quickly.

And that is the pickup sounds done!

Checkpoint Sounds

This section requires the Checkpoints module. If you do not have checkpoints in your game you can safely ignore this section.

Thanks to our setup in the Trigger scene, all we need to do here is select the TriggerSound node in the Checkpoint scene and drop our checkpoint sound into the inspector!

Switch Sounds

This section requires the Gates and Switches module. If you do not have switches in your game you can safely ignore this section.

And also just like our Checkpoint scene, all we need to add sound to our switches is to add the sound into the TriggerSound node in our Switch scene!
