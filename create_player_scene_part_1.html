Creating the Player Scene (Part 1)

This page will walk through setting up the player scene and assigning the player animations!

Choosing a Root Node

Continuing from our last tutorial, we should have a blank scene. In our scene tree Godot will be requesting we choose a Root Node.

A root node is the node from which all other nodes in a scene are descendants of. To keep with the tree metaphor, the root node is the roots of the tree that all the other nodes 'branch' off from. The root node provides the most basic functionality of a scene. For our player we will need to select a CharacterBody2D node. You may notice that is not an option in the scene tree right now, so we must select the Other Node option. This will bring up a menu to select from any of Godot's node types.

You can use the search bar at the top of the new node menu to quickly find nodes!

The root node for a scene also provides the name the scene will have when it is put into another scene. Due to this is is generally a good idea to rename your root node. You can do this by right clicking the node and selecting Rename. Let's rename our CharacterBody2D to Player.

To check out more details on node types, you can always right click a node in the scene tree and select 'Open Documentation'. Or to see all your options check out Godot's official documentation Here

Adding Sprites and Animations

You may notice that our scene in the viewport is still empty. This is because we haven't added any visuals to our player! In order to fix this we need to first add a new AnimatedSprite2D node as a child of our Player (CharacterBody2D) node by right clicking the Player node and selecting Add Child Node.

Navigate through the new node menu and select AnimatedSprite2D. Once it is added to the scene tree, select it and you will see it's details appear in the inspector. In order to add our sprites we must assign the node a Sprite Frames asset. We can do this by selecting the empty dropdown next to the sprite frames element and selecting New Sprite Frames.

Sprite frames are collections of 2D animations that we can switch between using code later on. We can select our new sprite frames in the inspector and a new tab will appear in the bottom panel: the sprite frames editor.

The sprite frames editor is split up into 4 major parts:

  1. Animation Controls

    Highlighted in blue, the animation controls allow you to (in order) create and delete animations, set an animation to play when the asset is loaded, set if an animation loops and its playback speed in frames per second.

  2. Animation Frame Controls

    Highlighted in green, the animation frame controls allow you to play and stop your animation in the veiwport (very useful for testing), add frames from textures or sprite sheets, copy and paste frames, insert blank frames, move and delete frames, and finaly set an individual frame's duration (relative to the animation's speed.)

  3. Animation List

    Highlighted in yellow, here is where you can see the list of animations in the sprite frames, as well as easily delete, select and rename them.

  4. Frame List

    Highlighted in orange, here you can see all the frames in the currently selected animation and in which order they will play.

For this project we will need four animations for our player: walking, jumping, falling and idle (standing still). But we don't have any textures yet! No worries, I have provided all resources necessary to complete every part of this course in the VGDC-2024 Assets file on your computer's desktop.

If you are not taking this course in person, you can download the assets Here

To import textures (or any other asset) into your Godot project is quite simple! Simply find the assets you wish to add (in this case, it is all the png files in the Getting Started file in our VGDC-2024 Assets folder) and drag them into your file system dock in Godot (bottom left corner beneath the scene tree.) Once they are imported, you can easily reorganize the assets in Godot.

Organizing your assets is incredibly important, especially as a project grows. You can create folders by right clicking in the file system dock. I usually have a folder for each type of asset, like textures, sounds, scripts, scenes etc. with subfolders for further structure. Do whatever organization strategy that makes the most sense to you, however!

Once we have imported our assets we can add our first animation! Select the default animation in the sprite frames editor on the bottom panel and rename it to idle. Select the Add Sprites from Sheet option in the animation frame controls. The icon looks like a three by three grid.

A sprite sheet is an image that has more than one frame or sprite on it.

A new menu will appear that will show your file system structure. Navigate through it to where you stored your player sprites and select the Green Dino Idle texture.

Another menu will appear prompting you to select frames, though before we can we must let Godot know how many sprites we have in the sprite sheet. We do this by using the horizontal and vertical elements in the top left corner of the menu. Horizontal is the number of sprites we have horizontally and vertically is the same but vertically (you can think of it like a grid.) Since we have four sprites in a horizontal line and only one row, we set horizontal to 4 and vertical to 1. We can then select out frames in the order we want them to play (in this case left to right.)

Once the frames are selected, you will see them in the frame list in the sprite frames editor. To add the rest of the animations, press the Add Animation button (the icon looks like a piece of paper with a plus sign), then repeat the steps you made for the idle animation.

Note that the jump and fall animations are 1 sprite long and walk is 6 sprites long instead of the idle animation's 4.

Once you have created all of the animations, change the walk animation's speed to 10 frames per second. Your sprite frames editor should look something like this:

You can select an animation and press play to see the animation play, though you may notice player is blurry. If you do, it's an easy fix. Go to the top bar of the Godot editor and select project, then the Project Settings option.

This will open up your project settings. Once there use the menu on the left to navigate to the Rendering/Textures section. Set the default texture filter to Nearest, then close the project settings.

You can use the Filter Settings bar to quickly search for settings!

You will now see nice crisp edges on your player's sprite! Go ahead and try out some different animation speeds. Once you are happy with them, you can move on to part 2 of making our character scene.
