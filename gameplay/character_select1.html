Character Select (Part 1)

IMPORTANT! This module requires the Main Menu and Global Script modules to be completed first!

Additionally, while not required, it would be easiest to complete whatever movement modules you want in your game before doing this one.

While a single, polished experience is often the best, sometimes it is nice to get some variety in your games. In this module we will create the ability for you to create multiple sets of player stats and visuals for players to choose.

Creating the PlayerData Resource

Our first step is to create a new Resource. We have already been working with resources, with sprite frames and tile sets being examples of some. A resource is a data container, and doesn't do anything on its own. That being said, they are great for modularity and can be connected to our nodes. To create a new one, right click a folder in your file system and create a new script. Make sure the script is inheriting Resource. Let's name the script PlayerData. This new resource will be used to store the pllayer stats for each of our playable characters.

In our new script we want to do a couple things. First, at the top of the script we will want to make the script a new class named PlayerData.

class_name PlayerData
extends Resource

Next, we want to add the variables in the player stats section of our player script. You can keep the current values as is, as that will be the default value for the player. We also want to make sure that they are all export variables.

@export var speed = 100.0
@export var acceleration = 800.0
@export var friction = 1200.0
@export var jump_velocity = -300.0
@export var gravity = 1.0
@export var terminal_velocity = 500.0

Note that if you have additional player stats from other modules, especially from movement modules, you will want to include them as well!

Finally, we want to add one more variable to our resource: a sprite frames so we can have different sprites for different characters. Let's create one with the following line:

@export var sprite_frames : SpriteFrames

Creating a new Character

Now that we have our PlayerData class, let's make an instance of it. Right click a folder in your file system (having a new Resource folder can be nice here), hover over create new and select resource.

In the menu that pops up, go ahead and search for our player data resource, select it, and give your new player data a name (I will name mine GreenPlayer). You will see the new resource pop up in the file system. To edit it we need to double click it. You will see it appear in the inspector.

For now I am going to keep the values where they are, as I am having this character be my default. But you may notice that the sprite frames are empty! To remedy this I am going to save my player's sprite frames in the file system and import them into the GreenPlayer resource. I can do this by going to my player scene, selecting the AnimatedSprite2D node, going to the sprite frames in the inspector, right clicking them and pressing save as.

Next, go back to GreenPlayer's inspector and drag our newly saved sprite frames into the sprite frames variable

And that is one character done! Of course, that means we still have more to go. But before we make more, let's give our player the ability to use these resources.

Modifying the Player

Let's open our player scene, and then its script. In the player stats section, go ahead and delete all the variables you copied into PlayerData earlier, and replace them with this line of code:

# Player Stats
@export var player_data : PlayerData

This line allows us to import one of our PlayerData resources. Though you may notice a lot of errors. This is because, well, we deleted a lot of variables being called. To remedy this, go through your player script and for each time one of the deleted variables was called, add "player_data." before it. So speed becomes player_data.speed. For further example, here is what our new apply_gravity function looks like (provided none of your other modules changed it):

func apply_gravity(delta):
	if not is_on_floor():
		velocity.y += player_data.gravity * delta
		velocity.y = min(velocity.y,player_data.terminal_velocity)

Note we are only adding the prefix to the variables we deleted. Local variables and reference variables are unchanged.

After clearing out the errors, you can test out if things work by selecting the player node and dragging your player data from the file system into the inspector.

Something else we need to do is make sure we update our sprite frames to the player_data's sprite frames. We can do this by calling our _ready function to set them like so:

func _ready():
	sprite.sprite_frames = player_data.sprite_frames

Go ahead and run your game to make sure everything works.

In the next part we will create some more characters, as well as create a menu where the player can select their character.
