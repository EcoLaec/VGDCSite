Character Select (Part 2)

In the last part we created the ability to store and set player data. In this one we will make some more characters and create a character select menu.

Creating More Characters

To make more characters, we can follow the same process we used to create the first one. I am going to make a Blue, Red and Yellow character, starting with creating a new PlayerData resource in the file system. Next, to handle the sprite frames it is easiest to duplicate the player sprite frames we already have, importing our new sprites and deleting the old.

You may need to import some additional sprites for your alternate characters. If you are not taking this course in person, you can download the assets Here

Finally, we can edit the PlayerData's stats in the inspector to make a unique feeling character. I am going to make my blue character light and floaty, my red character be heavy and snappy, and my yellow character fast but hard to control. Go ahead and mess around with your characters to get them just right. Don't forget to change the player data in your player scene to the character you are testing!

Creating a Character Select Menu

Now, we need a way for our players themselves to choose a character. We will do this by making a menu where they can choose before starting the first level. To keep things easy, let's duplicate our main menu scene as a base.

Let's delete the Play and Quit buttons, and remove the script from the main menu node, renaming our root node to Player Select. Next, create a new GridContainer node as a child of the VBoxContainer, and as a child of that, create four new buttons (or however number of characters you have.) Name each of the buttons after your characters.

Now to edit our nodes. First, select the MarginContainer and in the inspector navigate to theme overrides and change the margins to 75.

Next, change the text of the Title node to "Select your character". After that, select the GridContainer and set its columns to 2.

You can play around with all of these values to get your menu looking exactly how you'd like!

For each of the buttons you want to first set their alignments to expand. You can do this by selecting it and then opening the dropdown at the top of the viewport (the icon looks like a dotted line boxing in a green square.)

Next, go ahead and import the icons for each of the characters you are using from the Character Select folder in your VGDC Assets folder.

If you are not taking this course in person, you can download the assets Here

Select each of your buttons, dragging the respective icon in to the Icon variable, then under Icon Behavior set its alignment to center and Expand Icon to on.

Setting up the Scripts

Now to add functionality. Let's start by opening our Global script. In it let's create a new variable called player_data. We want this in the global script so we can store which player is selected across levels and scenes.

var player_data : PlayerData

Next go to our Player Select scene and add a new script to our Player Select node. Then, for each of our buttons, select it, go to the inspector and in the signals tab connect the pressed signal to our new script.

In our new script we want to create five variables. One to store the next scene, and four (or however many characters you have) to store the player data for our characters.

@export var green_player_data : PlayerData
@export var blue_player_data : PlayerData
@export var red_player_data : PlayerData
@export var yellow_player_data : PlayerData
@export var next_scene : PackedScene

Next, for each of the functions created by connecting the buttons' pressed signals, we want to set the global player_data variable to the respective PlayerData variable and move on to our next scene, like so:

func _on_green_pressed():
	Global.player_data = green_player_data
	get_tree().change_scene_to_packed(next_scene)

func _on_blue_pressed():
	Global.player_data = blue_player_data
	get_tree().change_scene_to_packed(next_scene)

func _on_red_pressed():
	Global.player_data = red_player_data
	get_tree().change_scene_to_packed(next_scene)

func _on_yellow_pressed():
	Global.player_data = yellow_player_data
	get_tree().change_scene_to_packed(next_scene)

Next, go to our player script. We want to add to our _ready function to have the global player_data variable imported to the player when the game starts.

func _ready():
	player_data = Global.player_data
	sprite.sprite_frames = player_data.sprite_frames

Finally, we need to assign some variables. Select the Player Select node and in the inspector drag your player datas into their proper places, as well as your first level. And last but not least, go to your main menu scene and change its next scene variable in the inspector to your player select scene.

And you are (finally) done! This is quite an intense module so congratulations! Go ahead and test out your characters and move on to another module!
