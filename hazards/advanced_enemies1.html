Advanced Enemies (Part 1)

IMPORTANT! This module requires both Simple Enemy modules to be completed first!

Up to this point our enemies have had pretty simplistic behaviors, mostly just walking back and forth. In this module, however, we will create two new enemies with far more complex behaviors: The Neura, a flying enemy that tracks the player around obstacles, and the Dilo, a small enemy that wanders about and spits acid.

Creating the Neura Scene

Let's start with the Neura. To create it, we will repeat the same steps we used for the trike in the Simple Enemy (Part 2) module, starting by creating a new inherited scene from our enemy scene.

Next, rename the Enemy node to Neura and select the AnimatedSprite2D and create a new sprite frames in the inspector. Then let's import the Neura move and death spritesheets. You can find them in the Advanced Enemies folder in your VGDC Assets folder.

If you are not taking this course in person, you can download the assets Here

Next, select the CollisionShape2D node and create a new rectangle shape in the inspector. Let's resize it to fit the sprite.

Finally, to make our Neura fly select the Neura node and go to the inspector. Let's set its speed to 75.0 and its gravity to 0.0.

At this point the Neura is just a faster version of the Ptero from the Intermediate Enemies module. To make actually be able to track the player we are going to use Godot's built in navigation. To do so we will add three new nodes. First a NavigationAgent2D node and a new Area2D node, which I will name Vision. Finally, create a CollisionShape2D node as a child of the Vision node.

Next, select the new NavigationAgent2D node. In the inspector we want to change both the Path Desired Distance and Target Desired Distance to 5.0. These variables dictate how close the agent needs to be to their "path" to consider reaching it.

I won't be going in depth into Godot's navigation system here, but if you're interested in learning more you can check out Godot's official documentation Here

Now to handle the Vision node. Select it and navigate to the inspector. We want to set it's layers to be blank and its masks to be just our player layer.

Finally, select the CollisionShape2D node under the Vision node. Create a new CircleShape2D in the inspector and size it to make the area in which the Neura will "see" the player. I made mine decently large:

Making the Neura Script

The scene may be done, but now we need to actually use the new nodes. Go ahead and disconnect the default Enemy script from the Neura node, then attach a new one. Let's name the new script neura and make it inherit from the Enemy class.

To start, we will want two new variables: target, which will be used to track the player, and a reference to our NavigationAgent2D node.

extends Enemy

@onready var navigation_agent = $NavigationAgent2D
var target : Node2D

Next, we are actually going to override our _physics_process function. We want to do this because the Neura doesn't move like any other enemy. There is a lot happening here, but I added comments to try and make things more digestable:

func _physics_process(delta):
	# Stop moving if dying or has no target
	if not is_moving or not target:
		velocity = Vector2.ZERO
		return
	
	# Target the player's location
	navigation_agent.target_position = target.global_position
	
	# Change direction based on movement
	if velocity.x != 0.0: sprite.flip_h = velocity.x < 0.0
	
	# Stop moving if the player has been reached
	if navigation_agent.is_navigation_finished():
		return
	
	# Calculate path and velocity required to reach the path
	var next_path_pos: Vector2 = navigation_agent.get_next_path_position()
	var new_velocity: Vector2 = global_position.direction_to(next_path_pos) * speed
	
	# Setting the velocity and moving
	if not is_moving: velocity = Vector2.ZERO
	velocity = new_velocity
	move_and_slide()

Finally we need to actually set the player as the target. We can do this by selecting the Vision node and navigating to the inspector under the signals tab. We want to connect the body_entered signal to a function called _on_player_spotted and the body_exited signal to _on_player_lost.

Back in the script we want to have the target variable set to the player when they are spotted, and removed when they are lost, like so:

func _on_player_spotted(body):
	target = body

func _on_player_lost(_body):
	target = null

Setting up the World Scene

There is one final problem, however. In order for our enemy to be able to navigate we need to set up a navigation region in our world scene! Go ahead and open up the world scene and create a new NavigationRegion2D node. Then, drag your GroundTiles to be a child of the new node (this will allow the navigation system to treat the level tiles as an obstacle).

Next, select the NavigationRegion2D node and in the inspector create a new navigation polygon.

Finally, with the navigation polygon selected in the inspector, you can click in the viewport to draw a region. Make sure to have the entire level (or at least the section you want the Neura to navigate) covered. You can finish the region by clicking your first point again to close the shape. After a brief delay, you should see a mesh of triangles appear to show the navigation region. If you don't, you can press the BakeNavigationPolygon button at the top of the viewport.

And that is the Neura done! Go ahead and add it to your level, check if it works, then move on to the Dilo!
