Health (Part 2)

Continuing from where we left off, I mentioned there was two issues with the health that had to still be solved. Those issues are 1) the ability for the player to not take damage if they stay within the collider of a single hazard and 2) the ability for the player to effectively go through some hazards we may not want them to (like the void collider or lava.) On this page we will fix both of those issues by adding the ability to Force a Respawn and Invincibility Frames.

Adding Invincibility Frames

Invincibility frames is a term used in game development for describing a time after the player takes damage where they cannot take damage again. We will use this to both make it so the player can't get hit in rapid succession as well as removing the ability for the player to safely sit within a hazard. To start, go to the player scene. Right click the player node and create a new Timer node. Let's name it InvincibilityTimer. In the inspector set it's wait_time to 1 second and set it's one_shot property to true.

Next open the player script. Select the InvincibilityTimer node and drag it into the reference variables section, start holding control and drop it in to get it as a reference variable.

# Reference Variables
@onready var sprite = $AnimatedSprite2D
@onready var invincibility_timer = $InvincibilityTimer
var respawn_point = Vector2.ZERO
var health = max_health

Next up we need to modify our take_damage function. Specifically, we will want to set the function to only run if the invincibility timer has no time left, and start the timer when the player does take damage, like so:

func take_damage():
	if invincibility_timer.time_left == 0.0:
		health -= 1
		invincibility_timer.start()
		if health <= 0:
			respawn()
			health = max_health

Finally, we want to check if the player is still touching a hazard when the invincibility timer runs out. We can do this by connecting the invincibility timer's timeout signal to our player script. Select the InvincibilityTimer node and in the signals tab in the inspector select the timeout signal.

In the new function, we want to disable and re-enable the collision shape for our HazardCollider. We do this to effectively reset it, alowing it to immediately send the signal for colliding with a hazard. We can do this like so: 

func _on_invincibility_timer_timeout():
	$HazardCollider/CollisionShape2D.disabled = true
	$HazardCollider/CollisionShape2D.disabled = false

There is problem one solved, but also it has made problem two even worse! Fortunately we have a solution.

Forcing Respawns

Many games have bottomless pits and lava that returns the player to solid ground, regardless of health. To start setting it up for our game, let's assign a new Group to our VoidCollider node in the world scene. Groups are effectively tags that we can check for objects to categorize them. Select the VoidCollider, then in the inspector select the the group tab. Press the plus symbol to create a new group. Let's name our group to be called ForceRespawn and set it to be global.

You can add this group to any hazard to force the player to reset if they collide with it. I would recommend adding it to the LavaTiles node if you have it.

Next, we need to actually use this group in our take_damage function in our player script. First add a new argument to the function called force_respawn with a required bool data type. Next, in the if statement where we check if the invincibility_timer has time left or force_respawn is true (we do to make sure it ignores invincibility so the player cannot slip through.) Finally, at the bottom of the function add if force_respawn: position = respawn_point.

func take_damage(force_respawn : bool):
	if invincibility_timer.time_left == 0.0 or force_respawn:
		health -= 1
		invincibility_timer.start()
		if health <= 0:
			respawn()
			health = max_health
		if force_respawn: position = respawn_point

Of course, we still need to make sure we are passing our new argument to our damage function. We can do this in our _on_hazard_collided function by adding a check for if the body or area are in the ForceRespawn group in our call_deferred method like so:

func _on_hazard_collided(_hazard):
	call_deferred("take_damage",_hazard.is_in_group("ForceRespawn"))

And that is health complete! Go ahead and test it out and then move on to another module!
