Sleepy Idle

In many games, if a player doesn't do anything for long enough, their character will do alternate idle animations, such as yawning or even taking a nap. We can add such an animation to make the game feel more lively.

Getting the New Assets

Before we can start adding the new animation, we need the new sprites! Go ahead and open up the Sleepy Idle folder in your VGDC Assets folder, find the appropriate tired and sleeping files and drag it into your file system.

If you are not taking this course in person, you can download the assets Here

Now we need to add two new animations to our player's sprite frames. Navigate to your player scene's AnimatedSprite2D then it's sprite frame. Create a new animation called tired and import the tired spritesheet as its frames, then repeat for the sleeping animation.

Note that the animation speed for sleeping is 2 fps! I also set the tired animation speed to 3 so that the player gradually slows down as they get more sleepy.

Adding the Idle Timer

Next we need to create a timer to keep track of how long the player has been idle. Navigate to the player scene tree and add a new node to our player node. Find the timer node, add it, and then rename it (I will name mine IdleTimer).

Editing the Player Script

Now we can finally add the functionality to our player script. First we need two variables: sleepiness (to keep track of which idle animation we should use) and a reference to our IdleTimer node. The former will be set to 0 by default:

# Reference Variables
@onready var sprite = $AnimatedSprite2D
@onready var idle_timer = $IdleTimer
var sleepiness = 0

Next, we want to add functionality to our _on_idle_timer_timeout function. Specifically, we want to increase our sleepiness variable each time it times out, to a maximum of 2. We can do this like so:

func _on_idle_timer_timeout():
	if sleepiness < 2: sleepiness += 1

Finally, in the update_animations function, we will start the IdleTimer if the player is idle (aka not moving), stopping the timer and resetting the sleepiness variable to 0 if they do move.

func update_animations(input_axis):
	# Tired/Sleeping
	if velocity == Vector2.ZERO:
		if idle_timer.is_stopped(): idle_timer.start()
	else:
		idle_timer.stop()
		sleepiness = 0
	
	# Update Sprite Direction
	if input_axis: sprite.flip_h = input_axis < 0
	
	# Grounded Animations
	if input_axis:
		sprite.play("walk")
	else:
		sprite.play("idle")
	
	# Aerial Animations
	if not is_on_floor():
		if velocity.y <= 0:
			sprite.play("jump")
		else:
			sprite.play("fall")

Though we still need to properly choose which idle animation to use! We can do this using a Match statement. A match statement is effectively a shortened version of a long chain of if/then statements, with a variable we put in (this case sleepiness) being compared to a bunch of values, with the first match having its code run and the rest skipped. Let's add it in where our sprite.play("idle") line is:

func update_animations(input_axis):
	# Tired/Sleeping
	if velocity == Vector2.ZERO:
		if idle_timer.is_stopped(): idle_timer.start()
	else:
		idle_timer.stop()
		sleepiness = 0
	
	# Update Sprite Direction
	if input_axis: sprite.flip_h = input_axis < 0
	
	# Grounded Animations
	if input_axis:
		sprite.play("walk")
	else:
		match sleepiness:
			1:
				sprite.play("tired")
			2:
				sprite.play("sleeping")
			_:
				sprite.play("idle")
	
	# Aerial Animations
	if not is_on_floor():
		if velocity.y <= 0:
			sprite.play("jump")
		else:
			sprite.play("fall")

The _: in the match statement is a default option, meaning it is always chosen if the previous values are false.
